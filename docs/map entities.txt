This is an extension of 'maps (pmf).txt'

- Entity names starting with 'player:' are reserved for player entities generated by the server.
- Signal names starting with 'unique:' are reserved for "unnamed" signals generated by the compiler.
- Signal inputs or outputs with empty names are ignored unless specified otherwise.
- For global or level entities with positions, the position is relative to the center of the center chunk. For chunk
  entities, the position is relative to the center of that chunk.

Entity table:
    <u32: Offset of <String table>>
    <u32: Entry count>
    <u32: Compressed size>
    <u8[]: Compressed [Entity table entry]...>
    <String table>
Static entity table:
    <u32: Entry count>
    <u32: Compressed size>
    <u8[]: Compressed [Static entity table entry]...>

Entity table entry:
    <u32:
        <8 bits: <Entity type> (must be higher than the previous entry's)>
        <24 bits: Entity count minus 1>
    >
    <Entity>...
Static entity table entry:
    <u32:
        <8 bits: <Static entity type> (must be higher than the previous entry's)>
        <24 bits: Static entity count minus 1>
    >
    <Static entity>...

Entity:
    <u32: Gamemodes bitmask>
    <u32: Size of following data>
    <<String>: Name>
    <u32: Custom property count>
    <Entity data>
    [Entity custom property]...
Static entity:
    <u32: Gamemodes bitmask>
    <u32: Size of following data>
    <u32: Custom property count>
    <Static entity data>
