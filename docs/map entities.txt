This is an extension of 'maps.txt'

- Entity names starting with 'player:' are reserved for addressing player entities.
- Signal names starting with 'unique:' are reserved for "unnamed" signals generated by the compiler.
- Signal inputs or outputs with empty names are ignored.
- For global or level entities with positions, the position is relative to the center of the center chunk. For sector
  entities, the position is relative to the center of that sector.

Entity: <<String>: Name> <u8: <Entity type>> <Entity data>

Entity data:
    <u8: Difficulties>
    <u16: Gamemodes>
    { <Headless entity data>
    | <Generic entity data>
    | <Scriptable entity data>
    | <Prop entity data>
    | ... }

Headless entity data:
    <<String>: Script resource path>
NPC entity data:
    <u8:
        <7 bits: 0>
        <1 bit: Active by default>
    >
    <float[3]: XYZ position>
    <float[3]: XYZ rotation>
    <<String>: Script resource path>
    <<String>: Model resource path>
    <<String>: Signal to activate>
    <<String>: Signal to deactivate>
    <<String>: Signal to toggle activation>

Prop entity data:
    <u8:
        <6 bits: 0>
        <1 bit: Collision>
        <1 bit: Shown by default>
    >
    <float[3]: XYZ position>
    <<String>: Entity to follow>
    <<String>: Bone to follow>
    <float[3]: XYZ hitbox radius>
    <<String>: Model resource path>
    <<String>: Animation>
    <<String>: Signal to show>
    <<String>: Signal to hide>
    <<String>: Signal to toggle visibility>

Decal entity data:
    <float[3]: XYZ origin>
    <float[3]: XYZ direction>
    <<String>: Texture>
    <u8[4]: RGBA color>

Setter entity data:
    <<String>: Signal>
    <<String>: Target entity (if empty, use initiator of signal)>
    <<String>: Property>
    <<String>: Value>
Point entity data:
    <u8:
        <7 bits: 0>
        <1 bit: Start at position 2>
    >
    <float[3]: XYZ position 1>
    <float[3]: XYZ position 2>
    <float[3]: XYZ velocity>
    <<String>: Signal to go to position 1>
    <<String>: Signal to go to position 2>
    <<String>: Signal to toggle positions>
Spawn area entity data:
    <float[3]: XYZ positon>
    <float[2]: XZ area radius>
    <u8[2]: XZ grid size>
    <<String>: Team (empty for any)>

String:
    <u32: Offset in string table>
